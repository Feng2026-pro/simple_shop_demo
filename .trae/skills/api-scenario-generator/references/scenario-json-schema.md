# 场景用例 JSON 格式（主输入）

与 [scenario-excel-spec.md](scenario-excel-spec.md) 列一一对应，供 `scenario_json_to_excel.py` 生成评审用 Excel。
本格式用于表达“单接口用例 + 多接口串联场景”，不负责直接生成执行用例。
默认用于**全量覆盖评审**：按模块覆盖 CRUD + 常见负向 + 边界值。

## 根结构

```json
{
  "cases": [
    { }
  ]
}
```

- `cases`：**必填**，数组；每项为一条评审对象（单接口或串联）。

## 场景对象字段

键名与 Excel 表头一致；缺失键按空字符串写入单元格。

| 键名 | 必填 | 说明 |
|------|------|------|
| 场景类型 | 是 | `单接口` 或 `串联` |
| 场景ID | 是 | 全表唯一 |
| 场景名称 | 是 | 一句话描述 |
| 测试点 | 是 | 本场景要覆盖的业务验证点 |
| 优先级 | 否 | 建议 `P0/P1/P2`，默认 `P1` |
| 前置条件 | 否 | 数据与环境 |
| 步骤 | 是 | 重点字段。建议编号多行描述接口串联过程（先调A，再调B） |
| 预期结果 | 是 | 场景级预期结果与关键校验点 |
| 备注 | 否 | |
| 预期状态码 | 否 | HTTP 预期，默认 200 |
| 路径参数 | 否 | JSON 对象**字符串**，如 `{\"userId\":\"1\"}` |
| 前置依赖 | 否 | 默认语义为 `无`；可填 `user_login` 等 |
| 依赖产出提取 | 否 | 与 runner 列一致，可留空 |
| 注入方式 | 否 | 如 `Header:Authorization:Bearer {token}` |

## 最小覆盖清单（默认要求）

对文档中的每个资源模块（如 `users/products/orders/inventory`）：

- 查询类：至少覆盖列表与详情；详情需包含“存在/不存在”。
- 创建类：至少覆盖创建成功与关键参数非法。
- 更新类：至少覆盖更新成功与非法更新（状态非法/字段非法等）。
- 删除类：至少覆盖可删成功与不可删约束（业务冲突或状态限制）。
- 数值字段：至少覆盖 0、负数、最小合法值（若文档可判断可再补超限）。

说明：

- `场景类型=单接口` 用于接口维度覆盖；`场景类型=串联` 用于业务链路覆盖。
- 生成顺序建议：先补齐单接口覆盖，再生成串联场景。

## 最小示例

```json
{
  "cases": [
    {
      "场景类型": "单接口",
      "场景ID": "ATOM_PROD_DEL_001",
      "场景名称": "删除商品-有库存时失败",
      "测试点": "DELETE /api/v1/products/{productId} 在 stock>0 时不可删除",
      "优先级": "P0",
      "前置条件": "目标商品库存大于 0",
      "步骤": "1) 调用 DELETE /api/v1/products/{productId}",
      "预期结果": "删除失败并返回业务错误",
      "备注": "对应文档删除约束"
    },
    {
      "场景类型": "串联",
      "场景ID": "SCN_CHAIN_001",
      "场景名称": "登录后下单并查询订单",
      "测试点": "验证登录态下的下单闭环",
      "优先级": "P0",
      "前置条件": "系统存在可售商品",
      "步骤": "1) 调用 /auth/login 获取 token\n2) 调用 /api/v1/orders/place 创建订单并记录 orderId\n3) 调用 /api/v1/orders/{orderId} 查询订单详情",
      "预期结果": "下单成功，且可查询到同一订单，状态与金额正确",
      "备注": "典型多接口串联主流程"
    }
  ]
}
```

## 与脚本的关系

1. 维护或生成 `*_scenario.json`（建议纳入版本库，根键为 `cases`）。
2. `python .../scenario_json_to_excel.py -i *_scenario.json -o *_scenario.xlsx`
